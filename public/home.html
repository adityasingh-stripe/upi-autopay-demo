Three Implementation Patterns

This demo shows PaymentIntent (server-side & client-side confirmation) and SetupIntent patterns for UPI autopay.

Server-Side PI

  • Uses PaymentIntent API
  • createPaymentMethod() on client
  • Server controls confirmation
  • Charges immediately + saves PM

Client-Side PI

  • Uses PaymentIntent API
  • confirmPayment() on client
  • Stripe.js handles confirmation
  • Charges immediately + saves PM
