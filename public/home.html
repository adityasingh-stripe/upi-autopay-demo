UPI Autopay Demo

Choose Your Confirmation Flow

Two Implementation Patterns

This demo shows both server-side and client-side confirmation patterns for UPI autopay (setup_future_usage).

Server-Side Confirmation

  • Uses createPaymentMethod()
  • Server controls confirmation
  • More control, flexible logic
  • Requires polling for UPI

Client-Side Confirmation

  • Uses confirmPayment()
  • Stripe.js handles confirmation
  • Simpler, less code
  • Auto-completes in test mode

Test Email: Configured via TEST_BILLING_EMAIL in .env (default: succeed_immediately@example.com)
